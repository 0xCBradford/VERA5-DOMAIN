# Vera5 UI Principles
Version: 1.0

---

# Core Philosophy

Vera5 interfaces should prioritize:

- operational clarity
- analyst efficiency
- low-fatigue interaction
- trust
- precision
- restrained technical identity

The UI should feel:

> like a trusted analyst utility, not a flashy cybersecurity startup.

---

# Primary Design Principle

## Precision Over Spectacle

Every UI decision should favor:
- readability
- speed
- clarity
- workflow efficiency

over:
- visual gimmicks
- animation-heavy interactions
- marketing theatrics
- unnecessary complexity

---

# Analyst-First Design

Vera5 exists for:
- SOC analysts
- CTI researchers
- DFIR operators
- malware analysts
- threat hunters

The UI must support:
- long sessions
- fast pivots
- dense information
- rapid context switching
- operational focus

---

# Low-Fatigue Interfaces

Avoid:
- bright whites
- high saturation
- excessive glow
- animation spam
- noisy layouts
- visual clutter

Favor:
- dark surfaces
- restrained contrast
- soft layering
- readable hierarchy
- intentional spacing

---

# Technical, Not Corporate

Vera5 should feel:
- modern
- technical
- OSS-native
- engineering-oriented

Avoid:
- generic SaaS templates
- sales-oriented layouts
- startup buzzword sections
- “enterprise AI platform” aesthetics

---

# Cyberpunk Restraint

Subtle futuristic energy is acceptable.

Excessive cyberpunk styling is forbidden.

Allowed:
- subtle amber accents
- restrained violet highlights
- layered dark surfaces
- minimal tactical glow

Forbidden:
- neon overload
- hacker green-on-black
- glowing grids everywhere
- sci-fi dashboard clutter
- gaming UI aesthetics

---

# Typography Principles

Typography should prioritize:
- readability
- hierarchy
- technical precision

Primary UI font:
- Inter

Technical font:
- JetBrains Mono

Accent/branding font:
- Space Grotesk

---

# Layout Philosophy

Interfaces should feel:
- spacious but dense
- layered but readable
- minimal but informative

Avoid:
- oversized hero sections
- excessive empty space
- cramped cards
- crowded dashboards

---

# Interaction Principles

Interactions should feel:
- fast
- intentional
- subtle
- responsive

Avoid:
- delayed transitions
- floating effects
- distracting motion
- unnecessary hover gimmicks

---

# Hover Card Philosophy

Hover cards are core to Vera5 identity.

They should feel:
- compact
- fast
- information-dense
- readable
- operational

Hover cards should prioritize:
- IOC clarity
- enrichment source visibility
- verdict readability
- rapid pivots

Vera5 should feel:

---

# Anti-Patterns

Vera5 must avoid becoming:

- a crypto dashboard
- a gamer UI
- a fake “Hollywood hacker” interface
- a marketing-heavy SaaS homepage
- a cluttered SIEM clone
- an animation showcase

---

# Forbidden UI Patterns

The Vera5 website and interfaces must avoid generic SaaS UI clichés.

Forbidden patterns include:
- oversized rounded “pill” badges
- floating chip-style labels
- giant CTA pills
- crypto-style stat cards
- glowing cyberpunk buttons
- fake dashboards
- excessive gradient blobs
- oversized whitespace-heavy layouts
- giant marketing sections
- startup-style feature bubble grids

Avoid:
- “Browser Extension” pills
- “AI Powered” pills
- “Next Gen Security” labels
- rounded floating tags everywhere

Preferred alternatives:
- clean inline labels
- subtle bordered metadata
- understated section headers
- restrained typography hierarchy
- technical layout structure

The UI should feel:
- operational
- mature
- structured
- engineering-oriented

NOT:
- startup-marketing-oriented
- crypto-inspired
- template-generated
- buzzword-driven

---

# Final Guiding Principle

Every interface should answer:

> Does this improve analyst workflow clarity without increasing cognitive load?
