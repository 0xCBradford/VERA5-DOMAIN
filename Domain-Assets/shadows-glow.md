# Vera5 Shadows & Glow System
Version: 1.0

---

# Philosophy

Shadows and glow effects should create:

- depth
- hierarchy
- focus
- subtle technical atmosphere

NOT:
- spectacle
- neon overload
- gaming aesthetics
- “hacker movie” visuals

The goal is:

> restrained operational depth.

---

# Core Principles

## Minimal By Default

Most Vera5 surfaces should rely on:
- layering
- spacing
- border contrast

NOT:
- giant shadows
- intense glows

---

# Shadow Philosophy

Shadows should feel:
- soft
- diffused
- lightweight
- understated

Avoid:
- harsh black shadows
- giant elevation jumps
- floating-card syndrome

---

# Recommended Shadow Layers

## LOW ELEVATION

Used for:
- cards
- hover panels
- tooltips

Example:
0 2px 8px rgba(0, 0, 0, 0.18)

---

## MEDIUM ELEVATION

Used for:
- modals
- dropdowns
- overlays

Example:
0 6px 18px rgba(0, 0, 0, 0.28)

---

## HIGH ELEVATION

Used rarely for:
- focused overlays
- major dialogs

Example:
0 12px 32px rgba(0, 0, 0, 0.35)

---

# Glow Philosophy

Glow effects should:
- guide attention
- indicate interaction
- reinforce state

Glow should NEVER:
- dominate the UI
- pulse aggressively
- create neon overload

---

# Approved Glow Colors

## Electric Amber

HEX:
#F5A524

Used for:
- active hover states
- primary actions
- focused interactions

---

## Terminal Teal

HEX:
#22C7A9

Used for:
- success states
- trusted enrichment
- operational confirmations

---

## Deep Violet

HEX:
#8B5CF6

Used sparingly for:
- advanced features
- graph overlays
- intelligence views

---

# Forbidden Glow Styles

Do NOT use:
- neon green glow spam
- giant blur halos
- rainbow gradients
- animated pulse storms
- full-page glow effects
- glowing borders everywhere

---

# Hover Glow Behavior

Hover states should:
- slightly brighten
- softly sharpen
- subtly elevate

Recommended:
- 3–8% brightness increase
- soft outer blur
- minimal opacity glow

Avoid:
- dramatic hover explosions

---

# Card Styling Philosophy

Cards should feel:
- grounded
- layered
- readable
- stable

Cards should NOT:
- float excessively
- look glassy everywhere
- resemble crypto dashboards

---

# Glassmorphism Rules

Light glassmorphism is allowed.

Heavy glassmorphism is forbidden.

Allowed:
- subtle transparency
- restrained blur
- soft layering

Forbidden:
- frosted-glass overload
- unreadable transparency
- blurry interfaces

---

# Border + Shadow Balance

Prefer:
- subtle borders
- minimal shadows
- layered contrast

Avoid:
- relying entirely on shadows
- ultra-thick borders
- aggressive outlines

---

# Modal Styling

Modals should:
- softly separate from background
- feel lightweight
- remain readable

Recommended:
- backdrop blur
- restrained elevation
- soft fade-in

---

# Dashboard Depth

Dashboards should use:
- layered surfaces
- clean grouping
- minimal depth hierarchy

Avoid:
- floating windows everywhere
- over-stacked cards
- heavy glow effects

---

# Visual Keywords

Shadows and glow should feel:

- restrained
- tactical
- modern
- technical
- layered
- mature
- low-fatigue
- operational

---

# Final Principle

Depth should communicate:

> structure and focus, not spectacle.
