# Vera5 Motion System
Version: 1.0

---

# Motion Philosophy

Motion should feel:
- restrained
- tactical
- smooth
- responsive
- modern

Motion should NEVER:
- distract analysts
- dominate the interface
- feel game-like
- feel theatrical

The objective is:

> subtle operational responsiveness.

---

# Core Principles

## Motion Supports Function

Animation exists to:
- reinforce interaction
- improve clarity
- communicate state changes
- guide attention subtly

Animation should never exist:
- purely for spectacle
- to impress users
- to mimic sci-fi interfaces

---

# Transition Timing

Recommended durations:

| Type | Duration |
|---|---|
| Hover | 120–160ms |
| Card transitions | 160–220ms |
| Modals | 180–240ms |
| Page fades | 200–300ms |

Avoid:
- slow cinematic transitions
- delayed interactions
- floaty movement

---

# Recommended Easing

Preferred:
- ease-out
- cubic-bezier(0.16, 1, 0.3, 1)

Avoid:
- bounce effects
- elastic effects
- exaggerated easing

---

# Hover States

Hover interactions should:
- slightly elevate
- subtly brighten
- softly sharpen borders

Avoid:
- dramatic movement
- large scaling
- aggressive glow pulses

Recommended:
- translateY(-1px to -2px)
- subtle border emphasis
- soft shadow increase

---

# Glow Rules

Glow effects are allowed ONLY:
- sparingly
- subtly
- intentionally

Allowed:
- amber hover glow
- subtle violet accent glow
- soft teal confirmation glow

Forbidden:
- neon green glow spam
- flashing glow effects
- rainbow gradients
- animated glow pulses

---

# Loading States

Loading indicators should feel:
- minimal
- technical
- clean

Prefer:
- subtle shimmer
- restrained spinners
- low-noise skeletons

Avoid:
- giant loading animations
- excessive pulsing
- dramatic progress effects

---

# Modal Behavior

Modals should:
- fade quickly
- scale subtly
- feel lightweight

Recommended:
- soft opacity fade
- slight scale transition
- backdrop blur

Avoid:
- heavy cinematic overlays
- slow fade-ins
- bouncing dialogs

---

# Dashboard Motion

Dashboards should feel:
- stable
- reliable
- calm

Avoid:
- constantly moving widgets
- animated backgrounds
- floating charts
- reactive motion overload

---

# Scroll Behavior

Scrolling should feel:
- smooth
- controlleds
- stable

Avoid:
- parallax overload
- scroll hijacking
- excessive scroll-triggered animations

---

# Accessibility

Motion must respect:
- reduced-motion preferences
- accessibility standards
- low-distraction workflows

Always support:
- reduced animation mode
- minimal transition fallback

---

# Anti-Patterns

Forbidden:
- hacker movie animations
- Matrix-style effects
- glowing grids
- animated scanlines
- terminal rain effects
- excessive particle systems
- gaming HUD motion

---

# Final Principle

Motion should communicate:

> operational responsiveness, not visual spectacle.
