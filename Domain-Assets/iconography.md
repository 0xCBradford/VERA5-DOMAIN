# Vera5 Iconography System
Version: 1.0

---

# Icon Philosophy

Vera5 iconography should Always feel:

- technical
- modern
- lightweight
- operational
- readable
- minimal
- professional

Icons should support:
- workflow clarity
- rapid recognition
- low-fatigue interfaces
- dense analyst environments

Icons should NEVER:
- become decorative clutter
- dominate layouts
- feel cartoonish
- feel game-like
- resemble “hacker movie” UI

The goal is:

> clear operational signaling.

---

# Preferred Icon Styles

Vera5 favors:

- outline icons
- soft rounded corners
- clean geometry
- medium stroke widths
- minimal visual noise

Preferred visual characteristics:
- 1.5px–2px strokes
- simple silhouettes
- restrained detail
- high readability at small sizes

---

### Lucide

- extension UI
- navigation

- buttons

- documentation

- consistent styling

## SECONDARY

### Tabler Icons

Used for:
- advanced workflows
- technical sections
- utility-heavy layouts

Why:
- engineering-oriented
- visually mature

---

## OPTIONAL

### Phosphor Icons

Used sparingly for:
- richer visual moments
- onboarding
- explanatory sections

Avoid overusing.

---

# Icon Usage Rules

## DO

- use icons to improve scan speed
- maintain consistent sizing
- align icons cleanly with text
- use icons functionally
- preserve minimalism

---

## DO NOT

- stack excessive icons
- use emoji-style icons
- use 3D icons
- use glossy icons
- use skeuomorphic icons
- use random mixed icon sets
- use inconsistent stroke weights

---

# Recommended Sizes

| Usage | Size |
|---|---|
| Small inline | 14px |
| Standard UI | 16px |
| Navigation | 18px |
| Cards | 20px |
| Major sections | 24px |
| Hero graphics | 32–48px |

Avoid oversized icons.

# Hover Card Icons

Hover cards should:
- remain compact
- use icons sparingly
- prioritize readability

Icons should assist:
- source recognition
- IOC categorization
- verdict clarity

Never:
- overload hover cards with visuals

---

# Animation Rules

Icons may:
- subtly brighten
- slightly elevate
- softly transition

Icons must NOT:
- spin continuously
- bounce
- pulse aggressively
- glow heavily

---

# Custom Icon Usage

Because Vera5 maintains a large internal icon library:

Custom icons should follow:
- existing Vera5 stroke widths
- existing spacing rhythm
- existing visual restraint

All imported icons should be normalized:
- size
- padding
- stroke thickness
- alignment

---

# Hero / Branding Icons

Hero visuals should feel:
- technical
- abstract
- tactical
- intelligence-oriented

Avoid:
- shields everywhere
- padlock spam
- skull imagery
- “anonymous hacker” clichés

Preferred:
- graph structures
- network abstractions
- intelligence overlays
- node relationships
- subtle enrichment visuals
