# Vera5 Screenshot & Demo Style Guide
Version: 1.0

---

# Purpose

This document defines how Vera5 screenshots, demos, recordings, and public visuals should appear.

The goal is:

- consistency
- professionalism
- credibility
- operational realism

---

# Screenshot Philosophy

Vera5 visuals should feel:

- technical
- authentic
- operational
- clean
- analyst-oriented

NOT:
- overly staged
- fake
- marketing-heavy
- cluttered

The objective is:

> real workflows presented clearly.

---

# Preferred Screenshot Content

Focus on:
- IOC enrichment
- hover cards
- analyst workflows
- pivoting
- enrichment sources
- clean extension behavior
- documentation clarity

---

# Avoid Fake Dashboards

Do NOT:
- create fake telemetry
- invent unrealistic SOC screens
- fabricate threat data
- create “Hollywood hacker” visuals

Authenticity matters more than spectacle.

---

# Browser Capture Style

Preferred:
- dark browser themes
- clean tabs
- minimal distractions
- realistic analyst environments

Recommended browsers:
- Chrome
- Brave
- Firefox

---

# Window Framing

Screenshots should:
- crop tightly
- focus on the workflow
- minimize dead space

Avoid:
- giant desktop captures
- cluttered taskbars
- irrelevant windows

---

# Terminal Screenshots

Terminal captures should:
- use dark themes
- use clean monospace fonts
- remain readable
- avoid excessive customization

Preferred fonts:
- JetBrains Mono
- IBM Plex Mono

---

# Recommended Terminal Themes

Preferred:
- dark neutral backgrounds
- restrained accents
- readable syntax colors

Avoid:
- rainbow terminals
- glowing hacker themes
- novelty terminal setups

---

# Demo Video Philosophy

Demo recordings should feel:
- calm
- professional
- workflow-oriented

Avoid:
- flashy transitions
- excessive music
- meme editing
- gaming-style pacing

Focus on:
- usability
- enrichment speed
- operational value

---

# Cursor Movement

Mouse movement should:
- feel deliberate
- remain smooth
- avoid frantic motion

Avoid:
- rapid cursor shaking
- unnecessary movement
- distracting highlighting

---

# Recording Environment

Preferred:
- dark mode
- clean desktop
- minimal distractions
- professional browser layout

---

# GitHub Screenshot Standards

Screenshots should:
- support README clarity
- show real functionality
- demonstrate workflow value

Recommended:
- annotated screenshots
- concise labels
- consistent sizing

---

# Annotation Style

Annotations should:
- remain minimal
- highlight workflow
- avoid clutter

Preferred:
- subtle arrows
- restrained outlines
- minimal labels

Avoid:
- giant red circles
- meme-style editing
- aggressive callouts

---

# Aspect Ratio Recommendations

| Content Type | Recommended Ratio |
|---|---|
| README screenshots | 16:9 |
| Hero screenshots | 16:9 |
| Social previews | 1.91:1 |
| Documentation images | variable |
| Hover card demos | focused crop |

---

# Color Consistency

All screenshots should align with:
- Vera5 color palette
- typography system
- motion restraint
- operational aesthetic

Long-term demos should showcase:
- IOC enrichment

NOT:

- exaggerated cyber theatrics

# Visual Keywords

Screenshots and demos should feel:

# Final Principle

Every screenshot should answer:

> Does this look like a real tool an analyst would actually trust and use?
