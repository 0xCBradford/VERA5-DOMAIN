# Vera5 Spacing System
Version: 1.0

---

# Philosophy

Spacing defines perceived professionalism.

Vera5 spacing should feel:
- deliberate
- structured
- breathable
- layered
- readable
- operational

The layout should avoid:
- cramped cards
- oversized empty sections
- inconsistent padding
- random spacing values

---

# Core Spacing Scale

Use an 8px spacing system.

Primary spacing values:

| Token | Value |
|---|---|
| XS | 4px |
| SM | 8px |
| MD | 16px |
| LG | 24px |
| XL | 32px |
| XXL | 48px |
| SECTION | 72px |
| HERO | 96px |

Avoid arbitrary spacing whenever possible.

---

# Layout Width

Recommended content widths:

| Area | Width |
|---|---|
| Reading Content | 720–820px |
| Dashboard Content | 1200–1400px |
| Hero Sections | 1100–1300px |
| Hover Cards | 320–420px |

---

# Section Spacing

## Landing Page Sections

Recommended:
- 72px top/bottom spacing
- 24–32px internal spacing

Avoid:
- giant SaaS whitespace
- compressed stacking

---

# Card Spacing

## Standard Cards

Padding:
- 20–24px

Gap between elements:
- 12–16px

Border radius:
- 12–16px

---

# Hover Cards

Padding:
- 14–18px

Gap between IOC elements:
- 8–12px

Must remain:
- compact
- dense
- readable

---

# Navigation Spacing

Top navigation:
- 20–28px horizontal padding

Nav item spacing:
- 16–24px

Avoid:
- oversized nav bars
- floating oversized menus

---

# Typography Spacing

## Headings

Heading bottom spacing:
- 12–20px

Paragraph spacing:
- 16–24px

Line height:
- 1.5–1.7

---

# Grid Philosophy

Prefer:
- clean alignment
- predictable rhythm
- layered spacing

Avoid:
- Pinterest-style chaos
- asymmetrical randomness
- aggressive masonry layouts

---

# Button Spacing

## Primary Buttons

Padding:
- 10px 18px
- 12px 22px

Gap between buttons:
- 12–16px

Avoid:
- giant CTA buttons
- oversized SaaS pill buttons

---

# Dashboard Rhythm

Dashboard areas should:
- breathe
- remain readable
- avoid visual overload

Recommended:
- layered cards
- grouped data
- clean section separation

---

# Mobile Spacing

Reduce spacing by ~15–20%.

Do NOT:
- collapse spacing entirely
- stack content too tightly

Mobile should remain:
- clean
- readable
- touch-friendly

---

# Final Principle

Spacing should create:

> calm operational clarity.
