# Vera5 Color System
Version: 1.0
Purpose: Primary visual identity and UI color governance for Vera5.

---

# Design Direction

Vera5 should feel:

- operational
- modern
- technical
- slightly cyberpunk
- dark-mode native
- analyst-oriented
- low-fatigue
- professional
- trustworthy
- minimal but memorable

The interface should avoid:

- generic SaaS blues
- washed-out grays
- neon overload
- “hacker movie” green-on-black
- excessive glow effects
- rainbow palettes
- over-saturated accents
- gaming aesthetics

The goal is:

> A mature operator console with subtle futuristic energy.

---

# Visual Identity Philosophy

Vera5 is not:
- a corporate SaaS dashboard
- a crypto project
- a gaming brand
- a flashy startup landing page

Vera5 IS:
- analyst tooling
- enrichment infrastructure
- operational workflow software
- local-first security tooling
- OSS-native cyber tooling

The palette should communicate:
- clarity
- confidence
- speed
- precision
- trust
- operational maturity

# Hover Card Styling

IOC hover cards should feel:
- compact
- high-signal
- layered
- readable
- fast

Recommended:
- soft blur backgrounds
- subtle transparency
- minimal shadows
- strong typography hierarchy
- low-noise spacing
- analyst-focused information density

---

# Recommended UI Ratios

## 70%
Dark neutral surfaces

## 20%
Text + utility layers

## 10%
Accent colors

Accent colors should be intentional, not dominant.

---

# Animation Philosophy

Animations should feel:
- smooth
- restrained
- tactical
- subtle

Avoid:
- flashy neon pulses
- large motion
- excessive transitions
- gamer UI effects

Recommended:
- 120–180ms hover transitions
- soft opacity fades
- subtle elevation changes
- minimal glow accents

---

# Glow Usage

Glow effects should be:
- extremely subtle
- blurred
- low-opacity
- reserved for active states

Recommended glow colors:
- amber
- violet
- teal

Avoid:
- bright green glows
- red glows everywhere
- rainbow lighting
- animated glow spam
